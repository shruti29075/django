

# 🔷 What is Django?

Django is a high-level Python web framework used to build secure, scalable, and fast web applications.

It follows the MVT (Model-View-Template) architecture and helps developers build websites quickly with less code.

Django was designed to make web development simple, clean, and secure.

---

# 🔷 Why Django is Needed?

In web development, building everything from scratch is time-consuming and risky.

Django is needed because:

- It reduces development time
- It provides built-in security
- It handles database easily
- It gives ready-made admin panel
- It follows organized architecture (MVT)

Instead of writing login system, admin panel, database connection manually, Django provides it automatically.

# 🔷 Django MVT Architecture (Diagram + Deep Explanation)

Django follows the **MVT (Model – View – Template)** architecture.

It divides the application into 3 main parts to keep code organized and clean.



## 📊 MVT Architecture Diagram

                User (Browser)
                       |
                       v
                  URL Dispatcher
                       |
                       v
                     View
                 (Business Logic)
                       |
            ---------------------
            |                   |
            v                   v
         Model              Template
      (Database)          (HTML Page)
            |                   |
            ---------------------
                       |
                       v
                 HTTP Response
                 (Shown to User)

----------------------------------------------------------------------------------


## 🔹 Step-by-Step Flow Explanation (Easy Language)

1️⃣ User sends request from browser  
Example: http://127.0.0.1:8000/

2️⃣ URL Dispatcher checks urls.py  
It decides which view should handle the request.

3️⃣ View runs logic  
View may:
- Get data from Model
- Process data
- Perform calculations

4️⃣ Model interacts with database  
Model:
- Defines database structure
- Fetches or saves data

5️⃣ Template displays data  
Template:
- HTML file
- Shows output to user

6️⃣ Response is sent back to browser  

----------------------------------------------------------------------------------


## 🔹 Understanding Each Component in Detail

### 🔵 1. Model (Database Layer)

- Represents database table.
- Defines fields (columns).
- Uses ORM (Object Relational Mapping).

Example:

class Student(models.Model):
    name = models.CharField(max_length=100)
    marks = models.IntegerField()

👉 This creates a Student table in database.

----------------------------------------------------------------------------------


### 🟢 2. View (Logic Layer)

- Contains main logic.
- Receives request.
- Returns response.

Example:

def home(request):
    return render(request, "home.html")

👉 View connects backend with frontend.

----------------------------------------------------------------------------------


### 🟡 3. Template (Presentation Layer)

- HTML file.
- Displays data using Django template language.

Example:

<h1>Welcome {{ name }}</h1>

👉 Shows dynamic data on webpage.

# 🔷 Professional Django Commands (Beginner to Advanced)

## 🔹 Install Django
pip install django

## 🔹 Create Project
django-admin startproject project_name

## 🔹 Go Inside Project
cd project_name

## 🔹 Create App
python manage.py startapp app_name

## 🔹 Run Server
python manage.py runserver

## 🔹 Make Migrations
python manage.py makemigrations

## 🔹 Apply Migrations
python manage.py migrate

## 🔹 Create Superuser (Admin Login)
python manage.py createsuperuser

## 🔹 Collect Static Files (For Deployment)
python manage.py collectstatic

## 🔹 Freeze Requirements
pip freeze > requirements.txt

4. Create project

6. Define models  
7. Run migrations  
8. Create views  
9. Create templates

11. Test project

# 🔷 How to Deploy (Beginner Friendly Steps)

1. Push project to GitHub
2. Create requirements.txt
3. Add ALLOWED_HOSTS in settings.py
4. Set DEBUG = False
5. Choose platform:
   - PythonAnywhere
   - Render
   - Railway
   - Heroku
6. Connect GitHub repository
7. Deploy
